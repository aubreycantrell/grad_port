StreamlineWellesley

Redesigning Wellesley’s service ticket system for clarity, transparency, and mobile access.
UX · HCI · Prototyping · 2024 Career Education SWE Bootcamp

Purpose

We reimagined Wellesley College’s existing Work Order application to improve usability, transparency, and trust. In a five-day design sprint, our team of four analyzed the legacy interface and created a mobile-first, brand-aligned prototype that simplifies how students submit, track, and manage maintenance requests.

What It Does

The redesigned platform enables students to easily create and track service requests, receive status updates, and contact facilities directly from any device. It consolidates scattered submission paths into one accessible system, aligned with Wellesley’s overall web identity.

What the App Addresses

Students previously faced fragmented forms, unclear ownership of tickets, and minimal mobile functionality. Our redesign addresses these through guided forms, transparent tracking, and responsive layouts that support use on the go.

How might we make the Wellesley Work Order system faster, clearer, and more trustworthy for students and staff?

Objectives

  • Improve usability through concise, guided forms.
  • Increase transparency with visible status, ownership, and timelines.
  • Ensure mobile-first access for quick submissions on the go.
  • Align the redesign with Wellesley’s brand and digital conventions.

Research

Research Goals

  • Identify usability pain points and workflow inefficiencies.
  • Evaluate Wellesley’s web design patterns for alignment and familiarity.
  • Define user priorities for submission, tracking, and communication.

Brand Alignment

We studied Wellesley’s official web design guidelines—spacing, color, typography, and navigation—to ensure continuity.

  • WCAG-compliant color palette and contrast ratios.
  • Consistent header, footer, and global navigation hierarchy.
  • Unified form and button styles to support quick recognition.

Existing Application Audit

We mapped the old user flow to identify friction points in submission and transparency gaps in tracking.

Design Prototype

User Flow Sketches

We restructured flows for the core actions — creating requests, viewing progress, and contacting facilities. Each interaction was designed around cognitive ease and mobile accessibility.

UI Design Choices

A modular card layout surfaces the most critical information first: request status, owner, and last update. Consistent color and iconography improve visual hierarchy and build user confidence.

Key Components

  • Create Request: Guided form for new submissions.
  • My Requests: Centralized list view for all tickets.
  • Search Requests: Permissions-aware lookup for transparency.
  • Contacts & Triage: Immediate access to emergency vs. non-emergency numbers.
  • FAQ & Feedback: Expandable help panel and user suggestions form.

Final Flows

The final responsive prototype, built in HTML, CSS, and JavaScript, demonstrates accessible, brand-aligned design optimized for simple submission.

Testing

We ran peer usability tests across devices to assess clarity, task completion time, and overall comprehension. Insights guided refinements to spacing, label clarity, and button affordance.

Challenges

Given the five-day sprint constraint, we balanced speed with usability—prioritizing mobile responsiveness, accessible color contrast, and rapid prototyping efficiency.

Reflections & What’s Next

  • Connect prototype to live ticketing API for real-time updates.
  • Expand accessibility (screen readers, keyboard navigation).
  • Conduct staff interviews to refine workflow alignment.

Takeaways

  • Mobile reality: Students report issues on the go—responsive design is essential.
  • Clarity builds trust: Visible ownership reduces anxiety and duplicate requests.
  • Consistency accelerates learning: Familiar Wellesley branding enhances adoption.
  • Collaboration under pressure: A compressed sprint reinforced iterative decision-making and visual communication skills.

Through this project, we learned how accessibility, brand alignment, and iterative design transform institutional tools into transparent, trustworthy experiences.
